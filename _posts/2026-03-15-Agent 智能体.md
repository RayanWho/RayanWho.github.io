# Agent（智能体）

> **核心概念**：会「自己拆步骤、自己调用工具、自己执行任务」的 AI。
>
> 如果普通大模型更像「你问一句，它答一句」的聊天助手，那 Agent 更像：**你给它一个目标，它会自己想办法把事情做下去。**

---

## 🎯 用最简单的话解释

### 普通 AI

```
你说：帮我写一段 Kotlin 代码
它就回你一段代码
到这里就结束了
```

### Agent

```
你说：帮我分析这个崩溃并修复

它可能会自己做这些事：
1. 读取日志
2. 找到报错点
3. 查看相关代码
4. 分析可能原因
5. 生成修复方案
6. 甚至直接改代码、跑测试
```

> 💡 **这就是 Agent 的味道了。**

---

## 📝 一句话定义

> **Agent = 有目标感 + 有步骤感 + 会用工具的 AI**

---

## 🆚 Agent 和普通 Prompt 的区别

| 类型 | 特点 | 示例 |
|------|------|------|
| **普通 Prompt** | 回答一个问题 | 「什么是 RAG？」→ 知识回答 |
| **Agent** | 完成一个任务 | 「读取这个目录下的文档，整理需求，生成技术方案，再输出开发任务清单」→ 连续任务 |

---

## 🧠 Agent 的核心能力

一个真正有用的 Agent，通常有 **4 个核心能力**：

### 1️⃣ 理解目标 🎯

知道你最终想完成什么。

```
✅ 写一个登录页面
✅ 生成短剧分镜
✅ 分析股票新闻
✅ 整理会议纪要并发邮件
```

### 2️⃣ 任务拆解 🔨

把大任务拆成小步骤。

例如「做一个 AI 知识库 App」，它会拆成：

```
1. 需求定义
2. 技术选型
3. 接口设计
4. 前端页面
5. 后端服务
6. 测试与部署
```

### 3️⃣ 使用工具 🛠️

它不只是「说」，还会「做」。

**可调用的工具：**

| 工具类型 | 示例 |
|----------|------|
| 🔍 搜索 | 搜索网页 |
| 📁 文件 | 读文件、写文件 |
| 💻 代码 | 执行代码 |
| 🔌 API | 调用外部 API |
| 🗄️ 数据 | 查询数据库 |
| 📧 通信 | 发邮件、发消息 |
| 📅 日程 | 创建日历事件 |
| 🎨 生成 | 调用模型生成图片/视频 |

### 4️⃣ 根据结果继续下一步 🔄

Agent 不是做一步就停，而是会根据上一步结果继续推进。

```
搜索没找到 → 换关键词
执行代码报错 → 分析错误再重试
文档不完整 → 补充提问或用现有信息继续
```

---

## 📖 一个最容易懂的例子

比如你做一个**「股票分析 Agent」**。

**用户说：**
> 分析一下英伟达最近是否值得关注

### Agent 可能会：

| 步骤 | 操作 |
|------|------|
| 1️⃣ | 搜索最新新闻 |
| 2️⃣ | 读取财报摘要 |
| 3️⃣ | 看市场情绪 |
| 4️⃣ | 提取利好和利空 |
| 5️⃣ | 生成结构化结论 |

### 输出结果：

```json
{
  "看多因素": ["...", "..."],
  "看空因素": ["...", "..."],
  "风险点": ["...", "..."],
  "观察价位": "...",
  "置信度": "80%"
}
```

> 💡 如果只是普通聊天模型，它可能只会凭记忆泛泛而谈。但 Agent 更像一个**「会干活的流程执行者」**。

---

## 🆚 Agent 和工作流的区别

这两个很容易混淆。

| 维度 | 工作流 (Workflow) | Agent |
|------|------------------|-------|
| **步骤** | 人提前把步骤写死 | 只给目标，AI 自己决定怎么做 |
| **流程** | 固定、稳定 | 灵活、自主决策 |
| **示例** | 读文档 → 总结 → 翻译 → 输出 Markdown | 「帮我根据这份需求文档，产出最合理的开发方案」 |

### 形象比喻

| 类型 | 比喻 |
|------|------|
| **Workflow** | 🏭 流水线 |
| **Agent** | 🤖 会自主行动的执行者 |

## 🧩 Agent 常见组成

一个 Agent 系统，一般包括这些部分：

```
┌─────────────────────────────────────────┐
│              Agent 系统                  │
├─────────────────────────────────────────┤
│  🧠 大模型                               │
│     负责理解、推理、决策                  │
├─────────────────────────────────────────┤
│  📝 Prompt / System Prompt              │
│     定义 Agent 的角色和规则               │
├─────────────────────────────────────────┤
│  💾 Memory（记忆）                       │
│     记录用户信息、进度、项目背景          │
├─────────────────────────────────────────┤
│  🛠️ Tools（工具）                        │
│     让 Agent 能操作外部世界               │
├─────────────────────────────────────────┤
│  📋 Planner（规划）                      │
│     决定下一步做什么                      │
├─────────────────────────────────────────┤
│  ⚡ Executor（执行器）                   │
│     真正调用工具、执行动作                │
└─────────────────────────────────────────┘
```

### 各组件说明

| 组件 | 作用 | 示例 |
|------|------|------|
| 🧠 大模型 | 理解、推理、决策 | LLM |
| 📝 Prompt | 定义角色和规则 | 「你是一个 Android 技术负责人」 |
| 💾 Memory | 记录上下文 | 用户是谁、之前做到了哪一步 |
| 🛠️ Tools | 操作外部世界 | 搜索、读文件、调用 API |
| 📋 Planner | 规划下一步 | 任务拆解、决策 |
| ⚡ Executor | 执行动作 | 调用工具、执行代码 |

## 🔥 为什么现在大家都在讲 Agent

因为单纯聊天已经不够了。

> **真正有商业价值的，是让 AI 不只是回答问题，而是直接参与完成工作。**

### Agent 可以做的：

| 领域 | 应用 |
|------|------|
| 💻 编程 | 自动写代码 |
| 📊 调研 | 自动做调研 |
| 📄 文档 | 自动整理文档 |
| 🎧 客服 | 自动客服处理 |
| 📈 数据 | 自动分析数据 |
| 🧪 测试 | 自动跑测试 |
| 📝 内容 | 自动生成内容并发布 |

**这就是 Agent 火起来的原因。**

## 🤖 Agent 的常见类型

### 单 Agent

一个 Agent 负责整个任务。

**适合：**
- 简单自动化
- 单一领域助手
- 个人工具

### 多 Agent

多个角色分工合作。

| 角色 | 职责 |
|------|------|
| 📋 Planner | 负责拆任务 |
| 🔍 Researcher | 负责查资料 |
| 💻 Coder | 负责写代码 |
| ✅ Reviewer | 负责审查结果 |
| ⚖️ Judge | 负责最终决策 |

> 💡 你一直在关注的 **openclaw 多 agent**，本质就是这个方向。

---

## 💼 把 Agent 理解成「AI 员工」

这个比喻很好懂：

| 类型 | 比喻 | 特点 |
|------|------|------|
| **普通大模型** | 🧑‍💼 被问才答的顾问 | 被动响应 |
| **Agent** | 👨‍💻 拿到任务后主动做事的员工 | 主动执行 |

> 💡 但它不是人，它还是模型驱动的系统。只是比普通聊天更接近「执行」。

---

## 👨‍💻 对开发者来说，Agent 到底意味着什么

如果你是 Android / 后端 / AI 应用开发者，Agent 意味着你在做的，不再只是：

```
❌ 调一下 LLM API
❌ 把回答显示出来
```

而是要开始设计：

```
✅ 任务怎么拆
✅ 工具怎么接
✅ 状态怎么传
✅ 结果怎么校验
✅ 失败怎么重试
✅ 多角色怎么协作
```

> 💡 **也就是说，重点从「聊天 UI」变成了「任务执行系统」。**

---

## 🧪 一个很实用的公式

> **Agent = LLM + Prompt + Memory + Tools + Planning + Action**

其中最关键的是：

> **它不只是生成文字，而是为了完成目标去行动。**

## 🎓 一句话总结

> **Agent，就是一个以大模型为核心，能够理解目标、拆解任务、调用工具并持续执行的 AI 执行体。**

再通俗一点：

| 类型 | 定位 |
|------|------|
| **普通 AI** | 🗣️ 回答者 |
| **Agent** | 🛠️ 做事者 |
