# 端侧 AI vs 云端 AI

> **核心区别**：AI 模型是跑在本地设备上，还是跑在远程服务器上。

---

## 📱 什么是端侧 AI

**端侧 AI**，就是 AI 直接运行在手机、电脑、汽车、摄像头、机器人等设备本地。

### 常见例子

- 📸 手机相册里本地识别人脸
- ⌨️ 输入法本地预测下一个词
- 📞 手机通话实时降噪、翻译
- 🎤 本地语音唤醒「你好，小爱/小艺/Siri」
- 📴 不联网也能跑的小模型问答

### 特点

| 特点 | 说明 |
|------|------|
| 🔒 隐私 | 数据不必上传云端 |
| ⚡ 速度 | 响应快，延迟低 |
| 📴 离线 | 离线可用 |
| 🔋 资源 | 更依赖设备算力、内存、电量 |

> 💡 **理解为**：「AI 装进设备里，让设备自己思考一点。」

---

## ☁️ 什么是云端 AI

**云端 AI**，就是 AI 运行在云服务器/数据中心，设备只是把请求发过去，再把结果拿回来。

### 常见例子

- 💬 调用 ChatGPT、通义千问、文心一言 API
- 🎨 云端图片生成
- 🤖 云端大模型问答
- 🎙️ 云端语音识别
- 📹 云端视频分析

### 特点

| 特点 | 说明 |
|------|------|
| 💪 能力强 | 模型可以很大，很强 |
| 📱 设备压力小 | 本地设备压力小 |
| 🌐 依赖网络 | 依赖网络 |
| ⏱️ 延迟 | 有网络延迟 |
| 📤 数据上传 | 数据通常需要上传到服务器处理 |

> 💡 **理解为**：「设备把问题交给远处的大脑去处理。」

---

## 🆚 两者对比

| 维度 | 端侧 AI | 云端 AI |
|------|---------|---------|
| 📍 运行位置 | 本地设备 | 云服务器 |
| 🌐 网络依赖 | 可离线 | 基本需要联网 |
| ⚡ 响应速度 | 通常更快 | 取决于网络 |
| 🔒 隐私性 | 更好 | 数据要上传 |
| 🧠 模型能力 | 通常较弱 | 通常更强 |
| 💰 成本结构 | 设备成本高 | 云算力成本高 |
| 🎯 适合场景 | 实时、隐私、离线 | 复杂推理、大模型、多用户服务 |

---

## 🎯 各自适合什么场景

### ✅ 适合端侧 AI 的场景

- 🔊 实时语音唤醒
- 📷 相机美颜/识别
- ⌨️ 键盘联想输入
- 🌐 本地翻译
- 🚗 车机、耳机、机器人
- 🔐 隐私敏感任务
- 📴 弱网/无网环境

### ✅ 适合云端 AI 的场景

- 💬 大模型聊天
- 📄 复杂文档理解
- 🎨 AI 绘图 / 视频生成
- 📚 企业知识库问答
- 👥 多人共享的智能服务
- 🔄 需要持续更新模型能力的产品

---

## 🔄 端云结合：主流路线

现实里，很少纯端侧或纯云端，更多是：

> **简单任务放端侧，复杂任务放云端。**

### 📱 手机 AI 助手示例

**端侧负责：**
- 🔊 唤醒词识别
- 🎤 本地语音降噪
- ⚡ 简单命令执行
- 📋 通知总结

**云端负责：**
- 📝 长文本总结
- 🤔 复杂问答
- 🔍 联网搜索
- 🎨 生成图片/代码

## 👨‍💻 对 Android / iOS 开发者的意义

### 端侧 AI — 你需要关心

| 关注点 | 说明 |
|--------|------|
| 📦 模型部署 | 模型怎么放到手机里 |
| 🛠️ 推理框架 | TensorFlow Lite, ONNX Runtime Mobile, MNN, NCNN, Core ML (iOS), MediaPipe |
| ⚡ 性能优化 | 怎么做性能优化 |
| 🎯 硬件加速 | 怎么调用 NPU / GPU |

### 云端 AI — 你需要关心

| 关注点 | 说明 |
|--------|------|
| 🔌 API 调用 | 如何调用 API |
| 📊 流式输出 | 流式输出处理 |
| 💬 会话管理 | 会话管理 |
| 💰 成本控制 | 成本控制 |
| 🔄 网络重试 | 网络重试 |
| 🔒 数据安全 | 数据安全 |
| 🧠 服务端能力 | RAG、Agent、工具调用等 |

## 📝 一句话总结

| 概念 | 定义 | 核心优势 |
|------|------|----------|
| **端侧 AI** | AI 在设备本地跑 | 实时、隐私、离线 |
| **云端 AI** | AI 在远程服务器跑 | 能力强、模型大、扩展性好 |

---

## 💡 核心思维

> 不是选择「端侧」或「云端」，而是**如何设计端云协同**。
>
> 好的移动端 AI 产品，应该：
> - 端侧处理实时、隐私、高频小任务
> - 云端处理复杂、重型、需要上下文的任务
> - 两者无缝切换，用户体验一致
