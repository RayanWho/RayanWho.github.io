> **总目标**：6 个月后，你应该具备这 5 项核心能力

---

## 🎯 5 项核心能力

| # | 能力 | 描述 |
|---|------|------|
| 1 | ☁️ 云端 LLM 集成 | 能把云端大模型接进 Android App |
| 2 | 📚 知识库 AI | 能做一个带知识库的 AI App |

---

## 💡 核心思维

> Android 官方已经把移动端 AI 的路径明确分成两大块：
>
> - **端侧 AI**：Gemini Nano + ML Kit → 本地、低延迟、隐私优先
> - **云端 AI**：Firebase / Gemini → 复杂推理和生成
>
> **核心思维**：不是只会调一个模型，而是会做**端云协同**。

## 第 1 个月：AI 工程基础认知

### 第 1 周：搭建你的移动端 AI 地图 🗺️

**这周不要急着做复杂功能，先把版图看明白。**

#### 你要搞懂 4 件事

- 🔹 什么是**端侧 AI**，什么是**云端 AI**
- 🔹 什么是**Prompt**，什么是**结构化输出**
- 🔹 什么是**RAG**
- 🔹 什么是**Agent**

#### 这周学习内容

- [ ] Android AI 总览
- [ ] Gemini Nano 是什么
- [ ] ML Kit 是什么
- [ ] 云端 LLM 和本地模型的区别
- [ ] 基本 Prompt 设计

#### 📦 这周产出

写一份你自己的笔记：**《移动端 AI 全景图》**

画一张图：`App UI → AI Service → 本地模型/云端模型 → 结果展示`

#### ✅ 本周标准

你能清楚回答：
- ❓ 为什么不是所有 AI 都该上云？
- ❓ 为什么移动端 AI 不只是聊天框？

### 第 2 周：做第一个"云端 LLM Android Demo" 💬

**这周目标非常明确：做一个最小可运行的 AI 聊天 Demo。**

不要追求产品化，只追求跑通链路：
1. 输入问题
2. 发请求
3. 返回回答
4. 展示在界面上

#### 📚 学习内容

- Kotlin 协程
- Retrofit / Ktor 网络请求
- SSE / 流式输出基础
- Chat 消息列表状态管理

#### 📦 这周产出

做一个最简单的 Demo：
- 一个输入框
- 一个发送按钮
- 一段流式回复
- 错误提示与 loading 状态

#### 💡 这周要建立的习惯

从一开始就把 AI 封装成独立层：

```
AiRepository
AiUseCase
ChatViewModel
```

> ⚠️ **不要把模型调用直接写进 Activity / Compose 页面里。**

### 第 3 周：学 Prompt 与结构化输出 📝

**这周你要从"能聊天"升级为"能让模型按格式工作"。**

#### 📚 学习内容

- system prompt / user prompt 的区别
- few-shot 基础
- JSON 输出
- 约束模型字段
- 错误兜底

#### 🛠️ 实战任务

做一个"文本处理工具页"，支持：
- 摘要
- 改写
- 生成标题
- 提取关键词
- 输出 JSON

#### 📦 这周产出

一个简单页面，输入一段文本后，可以返回：

```json
{
  "title": "...",
  "summary": "...",
  "keywords": ["...", "..."]
}
```

#### 💡 本周重点

> 以后你做 AI，不要只想着"让它说得像人"，而要想着"**让它给我可消费的结构化结果**"。

### 第 14 周：做端侧文本 AI Demo 🧪

#### 🛠️ 实战任务

做一个纯本地处理的小功能：
- 输入一段长文本
- 本地生成摘要
- 无网也能跑
- 显示耗时

#### 💡 本周重点

对比：
| 维度 | 云端 | 本地 |
|------|------|------|
| 延迟 | ⏱️ 较高 | ⚡ 低 |
| 隐私 | 🔓 需上传 | 🔒 本地处理 |
| 功能 | 💪 强大 | 📏 有限 |

> 这会让你第一次真正形成**端云协同**的工程直觉。

---

### 第 15 周：设计端云路由策略 🚦

这周不只是写功能，而是学"**怎么做决策**"。

#### 你要实现一个 Router

```
AiRoutePolicy
├── 文本 < 1000 字 → 优先本地
├── 敏感内容 → 优先本地
├── 复杂推理 → 走云端
├── 本地失败 → 降级云端
└── 云端超时 → 返回本地简化结果
```

#### 📦 这周产出

写一个 `AiRoutePolicy` 模块。

#### 💡 本周重点

> 这一步非常关键。以后你做移动端 AI，不应该只会"**调用模型**"，而应该会"**选择模型**"。

### 第 22 周：隐私与安全设计 🔒

**移动端 AI 很重要的一点是"你离用户隐私很近"。**

> ML Kit 很多能力强调本地处理，不上传输入数据；而端侧基础模型也明显服务于隐私和本地低延迟诉求。

#### 🛠️ 这周要做的事

检查你的项目：

- [ ] 哪些数据必须本地
- [ ] 哪些数据可以上云
- [ ] 上云前是否脱敏
- [ ] 是否有清缓存功能
- [ ] 权限申请是否最小化
- [ ] 是否向用户解释数据用途

#### 📦 本周产出

给你的项目写一份**《隐私与数据流说明》**。

## ⚠️ 最容易踩的坑

| 坑 | 描述 | 建议 |
|---|------|------|
| 🕳️ 坑 1 | 一开始就追求大而全 | 先做一个点，再做一条流，再做一个系统 |
| 🕳️ 坑 2 | 只会调接口，不会做产品 | 真正拉开差距的是：本地能力、UI 体验、系统集成、性能优化、失败兜底 |
| 🕳️ 坑 3 | 太早迷恋多 Agent | 先把**单 Agent + 工具调用**做稳，比什么都重要 |
| 🕳️ 坑 4 | 不做作品沉淀 | 学了很多但没有 GitHub 项目，转型效果会很差 |
