# Prompt 与结构化输出

> **核心概念**：Prompt 是你给 AI 的任务说明书，结构化输出是让 AI 按固定格式交答案。

---

## 📌 什么是 Prompt

**Prompt**，就是你给 AI 的「指令」。

### 常见例子

```
"帮我总结这篇文章"
"把这段话翻译成英文"
"你现在扮演一个产品经理，帮我写 PRD"
"请用 JSON 返回结果"
```

### 更准确的定义

> **Prompt = 任务说明 + 上下文 + 要求 + 输出格式约束**

---

## 🧩 完整 Prompt 的 5 个组成部分

### 1️⃣ 角色

告诉 AI 它现在是什么身份。

```
你是一个 Android 架构师
你是一个法律顾问
你是一个短视频编剧
```

### 2️⃣ 任务

告诉 AI 你要它做什么。

```
分析这段 Kotlin 代码的问题
给我写一个登录页面
提取这篇新闻里的关键信息
```

### 3️⃣ 上下文

给 AI 必要背景，不然它容易答偏。

```
这是给初学者看的
项目使用 Kotlin + Jetpack Compose
输出内容面向老板汇报
```

### 4️⃣ 约束

规定风格、长度、边界、格式。

```
不要太官方
用中文回答
控制在 200 字以内
不要输出解释，只给结果
```

### 5️⃣ 输出要求

明确最终想要什么格式。

```
用 Markdown 表格输出
返回 JSON
分成 3 点
每点一句话
```

---

## 🎯 Prompt 的本质

> 你可以把 Prompt 理解成：
>
> **你在给一个「很聪明但容易理解偏」的助手下任务。**

所以 Prompt 写得越清楚，AI 越容易输出你想要的结果。

### ❌ 差的 Prompt

```
帮我写一下登录功能
```

太模糊了。AI 不知道你要：
- 产品方案？
- UI？
- Android 代码？
- 接口设计？
- 测试用例？

### ✅ 更好的 Prompt

```
你是 Android 高级开发工程师。请用 Kotlin + MVVM + Retrofit，设计一个手机号验证码登录功能。
输出包括：
1. 模块划分
2. 接口定义
3. 关键代码骨架
4. 异常处理方案
```

这样结果会稳定很多。

## 📐 什么是结构化输出

**结构化输出**，就是：

> 不要让 AI 随便自由发挥，而是按你指定的固定结构返回结果。

### ❌ 散乱文本示例

```
这个股票总体还不错，公司基本面还行，风险也有，建议观察一下……
```

### ✅ 结构化输出示例

```json
{
  "stock_name": "英伟达",
  "sentiment": "bullish",
  "score": 8,
  "risks": ["估值偏高", "竞争加剧"],
  "suggestion": "可关注回调机会"
}
```

---

## 💡 为什么要结构化输出

### 1️⃣ 程序处理

AI 的结果可直接被代码解析：
- 📱 App 直接显示
- 🗄️ 后端直接入库
- 🤖 Agent 直接传给下一个步骤
- ⚙️ 自动化流程直接消费

### 2️⃣ 结果更稳定

自由文本容易：
- 忽长忽短
- 字段缺失
- 风格漂移

结构化输出更统一。

### 3️⃣ 方便校验

你可以检查：
- ✅ 有没有缺字段
- ✅ 字段类型对不对
- ✅ 枚举值合不合理

### 4️⃣ 适合多 Agent / 工作流

```
Agent 1（提取信息）→ Agent 2（打分）→ Agent 3（生成结论）
```

中间最好都用结构化数据传递，而不是一大段自然语言。

---

## 📋 结构化输出常见形式

### JSON（最常见）

最适合程序消费。

```json
{
  "title": "AI 手机助手",
  "platform": "Android",
  "priority": "high"
}
```

### Markdown 表格

适合人看，也比较规整。

| 模块 | 优先级 | 说明 |
|------|--------|------|
| 登录 | 高 | 核心入口 |
| 支付 | 高 | 商业闭环 |
| 设置 | 中 | 基础功能 |

### 固定字段列表

```
标题：
摘要：
风险：
建议：
```

### XML / YAML

一些工程场景也会用。

## 🔗 Prompt 和结构化输出的关系

| 概念 | 作用 |
|------|------|
| **Prompt** | 告诉 AI「做什么」 |
| **结构化输出** | 告诉 AI「怎么返回」 |

### 示例

```
你是一个财经分析助手。请分析下面这条新闻对股票的影响。
请严格按 JSON 输出，字段包括：
company、event、sentiment、impact_score、reason、risks
```

- 前半段 → **Prompt**
- 后半段 → **结构化输出要求**

## 📱 移动端开发实战案例

假设你做一个「AI 解析简历」的 App。

### ❌ 普通 Prompt

```
请帮我分析这份简历
```

返回可能很随意，不方便程序处理。

### ✅ 带结构化输出的 Prompt

```
你是一个 HR 助手。请分析下面的简历，并严格输出 JSON，字段包括：

- name: string
- years_of_experience: number
- skills: string[]
- strengths: string[]
- risks: string[]
- recommended_position: string
```

这样你的 Android / iOS / 后端就能直接解析。

## 🧪 好 Prompt 的实用公式

> **角色 + 任务 + 上下文 + 约束 + 输出格式**

### 完整示例

```
你是一个 Android 技术负责人。
请分析以下崩溃日志，找出最可能原因，并给出修复建议。
项目使用 Kotlin + MVVM。
回答要面向中级开发者，语言简洁。
请按 JSON 输出：error_type、root_cause、fix_suggestion、confidence
```

这就是一个比较完整、可控的 Prompt。

## 💻 对开发者最重要的理解

> 如果你想真正把 AI 用到产品里，不要只会「聊天式提问」，而要学会：
>
> **用 Prompt 控制任务，用结构化输出控制结果。**

因为真正落地时，最重要的不是「AI 会不会说」，而是：

> **AI 返回的结果能不能稳定地被你的程序接住。**
